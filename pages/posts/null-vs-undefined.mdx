Em JavaScript além dos tipos primitivos: number, string e boolean, existem os tipos **null** e **undefined**.

Muitos de nós temos bastante dificuldade em entender claramente esta diferença entre ambos, e comigo não foi diferente, mas hoje em dia uso bastante este conhecimento.

**null** significa um valor vazio ou inexistente. **nul**l é atribuído e explicitamente não significa nada.
**null** é do tipo object.

**undefined** significa que uma variável foi declarada, mas o valor dessa variável ainda não foi definido.
Diferente de **null**, undefined é do tipo **undefined**.

Este simples e poderoso conhecimento me ajuda a entender o que realmente está acontecendo no meu código quando preciso tratar estes dois tipos. 

```
  let test1 = null;
  console.log(test1); // null
  console.log(typeof test1); // object

  let test2;
  console.log(test2); // undefined
  console.log(typeof test2); // undefined

  null !== undefined  
  null == undefined

```
